Cách build APK

Push code lên GitHub

Đẩy toàn bộ code lên repository GitHub của bạn.

git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/your-username/PowerGuard.git
git push -u origin main

GitHub Actions tự động build

Khi push code, GitHub Actions sẽ tự động build APK. Kiểm tra tab "Actions" trong repository.

Tải APK

Sau khi build thành công, vào "Actions" > chọn workflow run > tải "Artifacts" (PowerGuard-debug hoặc PowerGuard-release).

Cài đặt trên thiết bị

Cài file APK lên thiết bị Android. Cấp các quyền cần thiết (Accessibility, Overlay).

Yêu cầu quyền

  • SYSTEM_ALERT_WINDOW - Hiển thị overlay mật khẩu trên các ứng dụng khác
  • Accessibility Service - Phát hiện khi menu nguồn được mở
  • FOREGROUND_SERVICE - Chạy dịch vụ bảo vệ trong nền
  • RECEIVE_BOOT_COMPLETED - Tự động khởi động khi thiết bị bật
  • Shizuku (tùy chọn) - Chặn sâu hơn các thao tác nguồn

Lưu ý quan trọng

Giới hạn Android Android không cho phép ứng dụng thông thường chặn hoàn toàn nút nguồn vì lý do bảo mật. Ứng dụng này hoạt động bằng cách:

  • Phát hiện: Sử dụng AccessibilityService để phát hiện khi menu nguồn (GlobalActionsDialog) xuất hiện
  • Đóng menu: Tự động thực hiện GLOBAL_ACTION_BACK để đóng menu nguồn
  • Xác thực: Hiển thị màn hình mật khẩu overlay toàn màn hình
  • Cho phép/Từ chối: Nếu mật khẩu đúng, người dùng có thể thao tác nguồn. Nếu sai, quay về màn hình chính

Shizuku Integration

Shizuku cung cấp quyền elevated (shell/root) cho ứng dụng mà không cần root thiết bị. Với Shizuku:

  • Ứng dụng có thể sử dụng các system API với quyền cao hơn
  • Cần cài đặt Shizuku và kích hoạt qua ADB hoặc root
  • Sau mỗi lần khởi động lại thiết bị (nếu không root), cần kích hoạt lại Shizuku qua ADB

Lưu ý: Ngay cả với Shizuku, Android vẫn có những giới hạn bảo mật cứng. Ứng dụng này cung cấp một lớp bảo vệ bổ sung nhưng không thể chặn 100% mọi cách tắt nguồn (ví dụ: giữ nút nguồn >10 giây sẽ force shutdown).
